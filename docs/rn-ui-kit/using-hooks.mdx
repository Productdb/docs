# Use React Native Core Hooks

ProductDB's React Native UI Kit provides multiple
[Hooks](https://beta.reactjs.org/reference/react). Hooks let you use different
React features from your components.

This provides seamless developer experience when integrating ProductDB's UI Kit in
your React native project. You can simply import the package from `ProductDBsdk/react-native-core`.

This page lists all the built-in Hooks in ProductDB.

- `<ProductDBProvider />`
- `<ProductDBUIProvider />`
- `useProductDBMeeting()`
- `useProductDBSelector()`

## `<ProductDBProvider />`

It is a simple context provider providing the meeting object to child
components.

```jsx
import { useProductDBClient } from '@ProductDBsdk/react-native-core';

function App() {
  const [meeting, initMeeting] = useProductDBClient();

  useEffect(() => {
    const init = async () => {
      initMeeting({
        authToken: '<auth-token>',
        defaults: {
          audio: true,
          video: true,
        },
      });
    };
    init();
  }, []);

  return (
    <ProductDBProvider value={meeting}>
      {/* Render your component */}
      <MyComponent />
    </ProductDBProvider>
  );
}
```

## `<ProductDBUIProvider />`

It is a simple context provider providing the ProductDB design system to child
components.

```jsx
import { useProductDBClient } from '@ProductDBsdk/react-native-core';
import { ProductDBUIProvider, ProductDBMeeting } from '@ProductDBsdk/react-native-ui-kit';

export default function App() {
  const [meeting, initMeeting] = useProductDBClient();

  useEffect(() => {
    const init = async () => {
      initMeeting({
        authToken: '<auth-token>',
        defaults: {
          audio: true,
          video: true,
        },
      });
    };
    init();
  }, []);

  return (
    <ProductDBProvider value={meeting}>
      <ProductDBUIProvider>
        <ProductDBMeeting meeting={meeting} />
      </ProductDBUIProvider>
    </ProductDBProvider>
  );
}
```

And to consume the context value, we provide two more hooks, each serving a
specific purpose.

These are:

- `useProductDBMeeting()`
- `useProductDBSelector()`

## `useProductDBMeeting()`

This hook essentially returns the `meeting` object you passed to the
`ProductDBProvider`.

The value returned doesn't re-render always whenever properties inside the
object change.

```jsx
import { useProductDBSelector, useProductDBMeeting } from '@ProductDBsdk/react-native-core';

function Meeting() {
  const { meeting } = useProductDBMeeting();

  /*
    use joinRoom() method or the setup screen component
    to actually enter the meeting
  */
  useEffect(() => {
    meeting.joinRoom();
  }, [meeting]);

  // show ProductDBMeeting
  return <ProductDBMeeting meeting={meeting} />;
}
```

## `useProductDBSelector()`

If you're familiar with Redux's
[useSelector](https://react-redux.js.org/api/hooks#useselector) hook, this hook
works in a similar way.

It allows you to extract data from the `meeting` object using a selector
function.

This hook will only cause a re-render when the actual data you returned for
changes.

Here is how you can get all the joined participants data:

```jsx
const joinedParticipants = useProductDBSelector(
  (meeting) => meeting.participants.joined
);
```

Refer to core [documentation for reference](/rn-core/reference/ProductDBClient) on
the various properties of the `meeting` object.

### Example

```jsx
import React from 'react';
import { View } from 'react-native';
import { useProductDBMeeting, useProductDBSelector } from '@ProductDBsdk/react-native-core';
import { ProductDBGrid, ProductDBButton, ProductDBText } from '@ProductDBsdk/react-ui-kit';

function Meeting() {
  const { meeting } = useProductDBMeeting();
  const roomJoined = useProductDBSelector((m) => m.self.roomJoined);

  if (!roomJoined) {
    return (
      <View>
        <ProductDBText>You haven't joined the room yet.</ProductDBText>
        <ProductDBButton onClick={() => meeting.joinRoom()}>Join Room</ProductDBButton>
      </div>
    );
  }

  return (
    <ProductDBGrid meeting={meeting} />
  );
}

export default Meeting;
```
